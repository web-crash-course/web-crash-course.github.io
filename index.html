What are web clients?

  • Web clients basically consist of HTML documents (content), JavaScript (behavior) and CSS (presentation) along with images, fonts, etc.
  • Web clients are served by web servers (e.g. Apache HTTP Server, NGINX, Apache Tomcat, Node.js).
  • Web clients run in web browsers (e.g. Chrome, Firefox, Edge, Safari)

Hyper Text Markup Language

HTML

HTML

  • HTML is a markup language for describing the structure of web documents (web pages)*
  • A basic HTML document looks like this:** 
    
<!DOCTYPE html>
<html lang="en">
<head>
    <title>Web client crash course</title>
</head>
<body>
    <h1>Welcome to a web client crash course</h1>
    <p>This is a <a href="demo.html">simple</a> sample.</p>
    <!-- this is a comment -->
</body>
</html>

HTML elements and tags

  • HTML documents consist of a tree of elements and text. Each element is denoted in the source by a start tag, such as <body>, and an end tag, such as </body>.
  • Tags have to be nested so that elements are all completely within each other, without overlapping: 
    
<p>This is <em>very <strong>wrong</em>!</strong></p>
<p>This <em>is <strong>correct</strong>.</em></p>
  • There are exceptions - so called void-elements e.g. 
    <input type="text">

HTML elements' attributes

  • Attributes control how the elements work and are placed inside the start tag.
  • Attributes consist of a name and a value, separated by an = character.
  • The value can remain unquoted if it doesn't contain space characters or any of " ' ` = > or < character.
  • Otherwise, it has to be quoted using either single or double quotes. The value, along with the = character, can be omitted altogether if the value is the empty string.

Attributes: examples


<!-- empty attributes -->
<input type=text disabled>
<input type=text disabled="">

<!-- attributes with a value -->
<input name=text maxlength=200>
<input name=text maxlength='200'>
<input name=text maxlength="200">

Cascading Style Sheets

CSS

CSS

  • CSS describes how HTML elements are to be displayed on screen*
  • CSS consists of rule-sets; each rule-set consists of selectors and declarations; a declaration includes a property and a value :** 
    
/* this is a CSS rule-set */
<selector1>, <selector2> { /* comma separated selectors */
  <property1> : <value1>;
  /* a property and a value make up a declaration */
  <property2> : <value2>;
  <property3> : <value3>
}

CSS example

In the example below p and div are selectors, background, font-weight and color are properties while red, bold and blue the properties' values.


p, div {
  background: red;
  font-weight: bold;
  color: blue
}

Basic CSS selectors

p All paragraphs
.class1 All elements with class1
#content The element with id="content"
p.class1 All paragraphs with class class1
p .class1 All elements with class class1 having a paragraph ancestor
.class1 .class2 All elements with class class2 having an ancestor with class class1
p.class1.class2 All paragraphs with both class1 and class2 classes

Block vs. inline (HTML) elements

display: block

  • displayed under each other; size defined by width, height, margin, padding
  • example: <div>, <section>, <p>

display: inline

  • displayed next to each other; size defined only by content size and margin-left, margin-right, padding-left, padding-right
  • examples: <button>, <input>, <a>, <span>

Flexbox

lays out, aligns and distributes space among items in a container

Flex container's properties

Flex Playground

display

defines the flex container


    .container {
        display: flex; /* or inline-flex */
    }

flex-direction

defines direction of flex item appearance


    .container {
        flex-direction: row | row-reverse | column | column-reverse;
    }

flex-wrap

allows items to wrap inside container


    .container {
      flex-wrap: nowrap | wrap | wrap-reverse;
    }

flex-flow

defines flex-direction and flex-wrap in one line


    .container {
        flex-flow: <‘flex-direction’> || <‘flex-wrap’>
    }

justify-content

defines alignment of items and usage of free space between them inside the container along the main axis


    .container {
      justify-content: flex-start | flex-end | center | space-between | space-around;
    }

align-items

defines alignment of container items along cross axis


    .container {
      align-items: flex-start | flex-end | center | baseline | stretch;
    }

align-content

aligns container lines (item rows) in the cross axis, affecting space between lines


    .container {
      align-content: flex-start | flex-end | center | space-between | space-around | stretch;
    }

Flex items' properties

Flex Playground

order

defines concrete order of special item inside a container


    .item {
      order: integer;
    }

align-self

overrides the cross axis alignment (align-items) for the individual item


    .item {
      align-self: auto | flex-start | flex-end | center | baseline | stretch;
    }

flex-grow

defines an ability to grow in proportion to free space


    .item {
      flex-grow: number; /* default 0 */
    }

flex-shrink

defines the ability to shrink, if no space available


      flex-shrink: number; /* default 1 */

flex-basis

defines the size of the item


      flex-basis: length | auto; /* default auto */

flex

defines flex-grow, flex-shrink and flex-basis in one line


    .item {
      flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]
    }

let declares a variable:


let foo = 123;

Objects

can be created using object literals:


const emptyObject = {};

Functions

  • are objects
  • can be created using function declaration:

function add(a, b) { // function declaration
    return a + b;
};

add(1, 2); // 3

Functions introduce the function scope

Functions can be anonymous (without name) and assigned to a constant or a variable:


const add = function (a, b) { // function expression
  return a + b;
};

add(1, 2); // 3

Functions can be written using the arrow function syntax:


const add = (a, b) => {
  return a + b;
};
// even shorter: the value is returned by default
const add = (a, b) => a + b;

Document Object Model (DOM)

  • Web browsers parse an HTML document, turning it into a DOM tree. A DOM tree is an in-memory representation of the document
  • The DOM tree can be manipulated from scripts (typically in JavaScript) in the page - through the Document interface
  • Each element in the DOM tree is represented by an object, and these objects have APIs so that they can be manipulated

The use of the document and element JavaScript APIs


  <form>
    <input id="firstName">
  </form>


// document is available in the browser's global scope
const inputElement = document.querySelector(
	'#firstName'); // #firstName is a CSS selector
// print the value of the input's value property
console.log(inputElement.value);

DOM interfaces: attributes vs. properties


  <form>
    <input id="firstName" value="John">
  </form>
					

const inputElement = document.querySelector('#firstName');
inputElement.value; // John - property value
inputElement.getAttribute('value'); // John - attribute value
					

inputElement.value; // Adam
inputElement.getAttribute('value'); // John

Event interface

represents an event which takes place in the DOM*


  <form>
    <input id="firstName">
  </form>
					

const inputElement = document.querySelector('#firstName');
inputElement.addEventListener( // add a listener to the input
    'keyup', // the event type
    event => // object which implements the event interface
            console.log(event.target.value));

Single Page Applications

SPAs

Classical web applications

  • The content of each web page is fully rendered on the server.
  • Each action requires a round trip to the server which contains presentation logic and application state.

Single Page Applications

  • The page's content is retrieved with a single page load.
  • The page does not reload any more.
  • Popular way to build rich web clients.

save(event: Event) {
  event.preventDefault();
  const formElement = event.target as HTMLFormElement;
  const firstNameInputElement = formElement
        && formElement.querySelector<HTMLInputElement>('#firstName');
  const lastNameInputElement = formElement
        && formElement.querySelector<HTMLInputElement<('#lastName');
  if (firstNameInputElement && firstNameInputElement.value
        && lastNameInputElement && lastNameInputElement.value) {
      console.log({firstName: firstNameInputElement.value,
                   lastName: lastNameInputElement.value});
  }
}

Angular Reactive Forms (1)

  • Import the ReactiveFormsModule in the AppModule
  • Create a backing object for the <form> element and initialize it: 
    
export class EmployeeDetailsComponent {
  employeeForm: FormGroup; // exposed to the template (form)

  constructor() {
    this.employeeForm = new FormGroup({
      // backing object for the first name input
      firstName: new FormControl('', // initially empty
          Validators.required), // is required
      lastName: new FormControl('', Validators.required)
    });
  }
}

Angular Reactive Forms (2)

  • Having the formGroup property we can make the save() method simpler: 
    
export class EmployeeDetailsComponent {
  employeeForm: FormGroup;
  // constructor here...

  save() {
    if (this.employeeForm.valid) {
      const firstName = this.employeeForm.get('firstName');
      const lastName = this.employeeForm.get('lastName');
      console.log({firstName, lastName});
    } else {
      console.error('Input values wrong...');
    }
  }
}
